# GOST Manager v2.0.1

GOST Manager v2.0.1 is a narrowly scoped setup compatibility hotfix for
supported Ubuntu hosts that use the standard `/etc/os-release` symlink to
`/usr/lib/os-release`.

## Fixed

- Public setup now accepts a safe regular `/etc/os-release` file.
- Public setup now accepts the standard relative or absolute symlink whose
  canonical target is `/usr/lib/os-release`.
- Public setup falls back to a safe regular `/usr/lib/os-release` when
  `/etc/os-release` is absent.
- OS validation still completes before release download, source activation,
  local installation, or any service operation.

## Security

The resolver uses only `/etc/os-release` and `/usr/lib/os-release` as trusted
canonical targets. It rejects missing or broken paths, symlink loops,
unexpected targets, non-regular files, group/world-writable metadata,
non-root-owned production metadata, oversized files, control characters, and
malformed required fields. The metadata file is parsed as data and is never
sourced, evaluated, or executed.

## Compatibility

- Ubuntu 22.04 LTS and Ubuntu 24.04 LTS remain the only supported releases.
- Direct Mode runners, profile management, traffic units, tunnel env files,
  firewall behavior, GOST installation, Server Stability, release archives,
  and transactional source/runtime ordering are unchanged.
- Monitoring remains outside the traffic path and its schema remains version
  4 with unchanged retention.

## Install

Latest stable release:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/WikiPanel/GOST-Manager/main/setup.sh)
```

Pinned release after v2.0.1 is published:

```bash
GOST_MANAGER_VERSION=v2.0.1 \
bash <(curl -fsSL https://raw.githubusercontent.com/WikiPanel/GOST-Manager/main/setup.sh)
```

The release workflow validates the exact version tag, runs the complete test
gate, creates the deterministic archive and SHA256 record, verifies both, and
publishes the non-draft stable Release. No v2.0.1 tag or asset is created
before review approval.
