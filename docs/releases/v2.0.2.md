# GOST Manager v2.0.2

GOST Manager v2.0.2 adds the production-hardened Iran Upstream Watchdog for managed Direct Mode profiles.

## Highlights

- Central per-profile health monitoring for every managed Iran profile.
- Disabled, Monitor Only, and Auto Protect modes.
- Exact default policy: checks every 2 seconds, declares an outage after 10 failures, and requires 10 successful checks plus hold and jitter before recovery.
- Auto Protect stops only the exact affected Iran GOST service and starts only a service whose stop it owns.
- Durable start and stop intents recover exactly once after daemon or host restart.
- Local Ping execution errors never count as upstream outages or trigger traffic actions.
- Manual service changes, maintenance, and operator overrides are preserved safely.
- Dedicated bounded 24-hour Watchdog event history and operational summaries.

## Compatibility and safety

Installation and upgrade leave every Watchdog profile Disabled and perform zero GOST traffic lifecycle actions. Direct Mode runners, env formats, firewall behavior, Server Stability values, and Monitoring schema 4 remain unchanged.

The release passed the full automated gate on Ubuntu 22.04 and Ubuntu 24.04. Controlled real-server tests validated Monitor Only and Auto Protect on both low-traffic and high-traffic profiles. During the high-traffic outage test, the exact affected service stopped at the configured threshold, its process socket descriptors fell to zero, host TCP allocation dropped substantially, the Iran host remained responsive, probing continued, and one qualified automatic recovery restored service.

A successful Ping proves host reachability, not the health of the remote SOCKS5 application itself.

## Upgrade

Use the normal GOST Manager latest-release setup flow. After upgrade, explicitly enable Auto Protect only for approved profiles through Upstream Watchdog menu option 12 or the administration CLI.
